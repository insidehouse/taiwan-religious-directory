# 廟通 MiaoTong — Pre-MVP 訪談腳本

> **用途**：帶去宮廟現場，與管理委員會 / 總幹事 / 廟公進行 60-90 分鐘深度訪談
> **目標**：驗證 3 個核心假設，決定是否投入 4 個月開發 MVP
> **時間**：2026 年 2-3 月（農曆新年旺季剛結束，廟方較有空）

---

## 訪談前準備

### 預約方式

1. **電話預約**（最有效）：
   - 「您好，我們在做宮廟數位化的研究，想請教貴廟管理上的經驗，大約一個小時，不知道主委或總幹事方便聊聊嗎？」
   - 避免說「推銷軟體」、「做產品」——會被當業務拒絕
   - 帶伴手禮（水果禮盒 NT$300-500）表示尊重

2. **選廟標準**：
   - 中型規模（信徒 500-3,000 人，有專職或兼職總幹事）
   - 有官方 LINE / Facebook 粉專（表示有基礎數位意識）
   - 高雄 / 台南優先（宗教組織密集，交通方便）
   - 不同神明類型各挑 1-2 間（媽祖、王爺、土地公、佛祖）

3. **目標受訪者**：
   - 最佳：**總幹事**（實際做事的人）
   - 次佳：**主委**（做決定的人）
   - 補充：**資深志工**（60+ 歲，用來做操作實測）

### 攜帶物品

- [ ] 此腳本（列印紙本）
- [ ] 筆記本 + 筆（不要只用手機記——廟方會覺得你在滑手機）
- [ ] 手機（錄音用，要先取得同意）
- [ ] LINE LIFF 原型 Demo（Figma 點擊原型，預載在手機上）
- [ ] 伴手禮
- [ ] 名片或簡單自我介紹單張

## Part 1：破冰與建立信任（10 分鐘）

> **目標**：讓對方放下戒心，願意說真話

### 開場白

「謝謝 [稱謂] 撥時間，我自己對宮廟文化很有興趣，最近在研究台灣宮廟在管理上遇到的困難，想多聽聽您們的經驗。今天不是來推銷什麼，純粹是來學習的。」

### 暖場問題

- 「貴廟主祀哪位神明？建廟多久了？」
- 「您在廟裡服務多久了？是什麼契機開始參與的？」
- 「今年過年（點燈/安太歲）的狀況怎麼樣？忙不忙？」

> **🎯 聆聽重點**：注意他們提到的任何「痛」或「抱怨」，即使是無意的

### 錄音同意

「為了怕我自己筆記記不完整，可以用手機錄音嗎？純粹自己做筆記用，不會公開。」

## Part 2：現況理解（20 分鐘）

> **目標**：深入了解目前的管理方式和真實痛點
> **原則**：先問「怎麼做」，再問「有什麼困擾」。不要引導。

### 2A：信徒管理

| # | 問題 | 記錄欄 |
|---|------|--------|
| 2A-1 | 「你們有沒有信徒的名冊或資料？是紙本的還是電腦的？」 | |
| 2A-2 | 「大概有多少位比較固定的信徒？（每年都會來點燈或捐款的）」 | |
| 2A-3 | 「如果有信徒搬家或換電話，你們怎麼更新資料？」 | |
| 2A-4 | 「有沒有遇到『找不到某位信徒資料』的情況？」 | |

> **🎯 觀察重點**：他們說「名冊」時，看看能不能請他們拿出來看。是手寫本？Excel？活頁夾？

### 2B：點燈 / 安太歲流程

| # | 問題 | 記錄欄 |
|---|------|--------|
| 2B-1 | 「今年安太歲/點光明燈大概有多少人登記？」 | |
| 2B-2 | 「登記的流程是怎麼進行的？（現場填單？電話？LINE？）」 | |
| 2B-3 | 「費用怎麼收？（現金？匯款？）一盞燈多少錢？」 | |
| 2B-4 | 「登記資料怎麼整理？誰負責？大概花多少時間？」 | |
| 2B-5 | 「有沒有遇過『信徒覺得自己有登記但其實沒有』的情況？」 | |
| 2B-6 | 「過年前最忙的時候，登記作業會不會出錯或手忙腳亂？」 | |

> **🎯 黃金問題（2B-6）**：如果對方開始抱怨春節忙亂，代表這是真實痛點

### 2C：財務管理

| # | 問題 | 記錄欄 |
|---|------|--------|
| 2C-1 | 「廟裡的帳目是誰在記？用什麼方式記？」 | |
| 2C-2 | 「每個月/每年大概要處理多少筆捐款記錄？」 | |
| 2C-3 | 「捐款收據怎麼開？有信徒要報稅用的嗎？」 | |
| 2C-4 | 「監事會或信徒大會看帳的時候，報表怎麼準備？」 | |
| 2C-5 | 「有沒有遇過帳目對不起來的狀況？」 | |

> **🎯 敏感話題**：財務問題廟方可能不想深談，不要追問。他們願意講多少就聽多少。

### 2D：現有數位工具

| # | 問題 | 記錄欄 |
|---|------|--------|
| 2D-1 | 「廟裡有用什麼電腦軟體或手機 App 來管事情嗎？」 | |
| 2D-2 | 「有 LINE 官方帳號嗎？主要拿來做什麼？」 | |
| 2D-3 | 「有試過用手機收香油錢嗎？（LINE Pay、台灣 Pay 之類的）」 | |
| 2D-4 | （如果有用數位工具）「用起來覺得怎麼樣？有什麼不方便的？」 | |
| 2D-5 | （如果沒用）「有沒有想過用什麼工具來幫忙管理？為什麼沒有用？」 | |

> **🎯 2D-5 是關鍵**：如果答案是「不需要」而非「找不到好工具」，代表需求不存在

## Part 3：痛點排序（10 分鐘）

> **目標**：確認哪些問題是「真的痛」，而非研究者自己的投射

### 卡片排序法

準備以下 6 張紙卡（事先寫好），請受訪者排出「最困擾 → 最不困擾」：

```
[A] 點燈/安太歲登記很繁瑣，容易出錯
[B] 信徒資料散落各處，不好找
[C] 帳目管理費時，對帳麻煩
[D] 通知信徒活動或法會不方便
[E] 想讓年輕人更容易參與廟裡事務
[F] 想要更透明的財務報表給信徒看
```

**記錄排序結果**：

```
最困擾 → ____  ____  ____  ____  ____  ____ ← 最不困擾

補充說明（受訪者的原話）：
_____________________________________________
_____________________________________________
```

> **🎯 判讀標準**：
> - 如果 [A] 排前三 → 點燈 Hook 有市場
> - 如果 [A] 排最後 → 核心假設 D2 可能有問題
> - 如果受訪者說「這些都不是問題」→ 紅燈警訊

## Part 4：概念測試（15 分鐘）

> **目標**：展示產品概念，觀察反應
> **原則**：先描述情境，再展示 Demo。不要解釋太多，觀察他們自己的反應。

### 4A：情境描述

「假設有一個工具，可以讓信徒在 LINE 上登記點光明燈，填好姓名、生辰，直接用 LINE Pay 付款，廟裡這邊會自動收到資料，不用再一張一張抄。燈還是在廟裡幫他們點，點好之後可以傳一張照片給信徒看。」

**停頓，觀察反應。然後問：**

| # | 問題 | 記錄欄 |
|---|------|--------|
| 4A-1 | 「聽起來您覺得怎麼樣？」（開放式，不引導） | |
| 4A-2 | 「您覺得你們廟的信徒會願意用嗎？」 | |
| 4A-3 | 「管理上會比現在方便嗎？還是差不多？」 | |
| 4A-4 | 「有沒有什麼讓您擔心的地方？」 | |

> **🎯 觀察**：
> - 眼睛亮起來 → 正面訊號
> - 皺眉 / 猶豫 → 追問原因
> - 「很好啊但是…」→ 「但是」後面才是真話

### 4B：Demo 展示

拿出手機，展示 LINE LIFF 原型（Figma 點擊原型）：

「我做了一個很簡單的示意，您看看這個畫面…」

展示流程：
1. 信徒端：LINE → 選燈型 → 填資料 → 付款 → 收到確認
2. 管理端：LINE → 今日登記清單 → 信徒資料 → 本月統計

**展示後問：**

| # | 問題 | 記錄欄 |
|---|------|--------|
| 4B-1 | 「這個操作起來會不會太複雜？」 | |
| 4B-2 | 「您覺得字夠不夠大？看得清楚嗎？」 | |
| 4B-3 | 「有沒有哪個步驟您覺得多餘的或不需要的？」 | |
| 4B-4 | 「您覺得缺了什麼？有沒有什麼功能一定要有的？」 | |

> **🎯 4B-4 是金礦**：如果 5 間廟都提到同一個缺少的功能，那就是真需求

## Part 5：付費意願測試（10 分鐘）

> **目標**：測試真實的付費意願，不只是「覺得不錯」
> **原則**：逐步推進，從模糊到具體，最後要求承諾

### 5A：價值認知

| # | 問題 | 記錄欄 |
|---|------|--------|
| 5A-1 | 「如果有這樣的工具，您覺得能幫廟裡省多少時間？」 | |
| 5A-2 | 「目前在管理上每個月大概花多少錢？（紙張、印表機、人力、電話費…）」 | |

### 5B：價格反應

「這種工具如果做出來，基本功能免費，進階功能（像是群發通知、財務報表匯出）每個月收費。」

| # | 問題 | 記錄欄 |
|---|------|--------|
| 5B-1 | 「您覺得一個月多少錢是合理的？」（讓他們先說） | |
| 5B-2 | 「如果是一個月 NT$499，您覺得值不值？」 | |
| 5B-3 | 「這個費用是誰決定的？需要開會嗎？大概要多久？」 | |

### 5C：承諾測試（最關鍵）

> **⚠️ 這是整場訪談最重要的問題。不要跳過。**

依序問以下三個問題，每個問題代表不同程度的承諾：

**Level 1 — 口頭意願：**
> 「如果我們三個月後做出來，免費讓你們試用一個月，您願意試試看嗎？」

```
回答：□ 願意  □ 要再考慮  □ 不需要
```

**Level 2 — 時間承諾：**
> 「試用的時候我們會來廟裡幫忙設定，大概需要半天時間，您方便嗎？」

```
回答：□ 方便  □ 要看時間  □ 不太方便
```

**Level 3 — 金錢承諾（關鍵中的關鍵）：**
> 「如果試用完覺得好用，下個月開始每月 NT$499，您這邊能做主嗎？還是需要委員會決議？」

```
回答：□ 我能做主  □ 需要委員會  □ 可能不行
預計決策時間：_____________
```

> **🎯 判讀標準**：
>
> | Level 1 | Level 2 | Level 3 | 判定 |
> |---------|---------|---------|------|
> | 願意 | 方便 | 能做主 | 🟢 **強信號** — 列為首批客戶 |
> | 願意 | 方便 | 需委員會 | 🟡 **中信號** — 有機會但決策慢 |
> | 願意 | 要看時間 | — | 🟠 **弱信號** — 禮貌性回應 |
> | 要再考慮 | — | — | 🔴 **無興趣** — 不要自我欺騙 |

## 附錄 A：常見異議與回應

> 訪談時如果遇到以下反應，可以參考回應方式。
> **注意：目的是理解，不是說服。如果對方真的不需要，記錄下來就好。**

| 廟方說 | 你的回應 | 真實含義 |
|--------|---------|---------|
| 「我們用紙本就好了」 | 「了解，紙本用了多久了？有沒有遇過什麼不方便的？」 | 可能真的不需要，也可能是抗拒改變 |
| 「信徒都老人家，不會用」 | 「那平常怎麼跟信徒聯絡？用電話嗎？」 | 探測 LINE 使用狀況 |
| 「我們沒有預算」 | 「現在管理上有花到什麼費用嗎？紙張、列印之類的？」 | 了解隱藏成本 |
| 「要開會才能決定」 | 「通常委員會多久開一次？如果是小金額他們會比較快決定嗎？」 | 了解決策流程 |
| 「之前有人來推銷過」 | 「是什麼產品？後來為什麼沒用？」 | 了解競品和失敗原因 |
| 「怕資料外洩」 | 「這個擔心很合理。您覺得什麼樣的保護方式您會比較放心？」 | 了解信任門檻 |

## 附錄 B：訪談 Do's & Don'ts

### Do's ✅

- **多聽少說**。你說話的時間不應超過 30%
- **記錄原話**。他們的用詞比你的摘要更有價值
- **追問「為什麼」**。表面答案背後才是真需求
- **觀察環境**。辦公桌上的紙本、牆上的公佈欄、功德箱旁有沒有 QR Code
- **尊重對方的時間**。說好 60 分鐘就 60 分鐘，除非對方主動延長
- **保持中立**。不要急著推銷概念或反駁疑慮

### Don'ts ❌

- **不要問引導式問題**：「您覺得線上點燈很方便對吧？」→ 會得到假陽性
- **不要用科技術語**：「SaaS」「LIFF」「PWA」「雲端」→ 用「手機上的工具」
- **不要在意冷場**。沉默 5 秒很正常，對方在思考
- **不要爭辯**。如果對方說「不需要」，不要試圖說服
- **不要承諾做不到的事**：「我們一定會做 XXX 功能」→ 改說「這是很好的建議，我記下來」
- **不要批評現有做法**：「紙本很落伍」→ 會讓人防禦

## 附錄 C：找廟方式

### 線上搜尋

1. **Google Maps**：搜尋「高雄 宮廟」「台南 媽祖廟」
   - 篩選條件：有 Google 評論 > 50 則（代表有一定規模）
   - 有官網或 Facebook → 數位意識較高

2. **Facebook 搜尋**：搜尋「XX宮」「XX廟」粉專
   - 有定期發文 → 有人在經營
   - 有 LINE 官方帳號連結 → 更好

3. **內政部全國宗教資訊網**：https://religion.moi.gov.tw
   - 可查詢已登記宮廟
   - 有地址和聯絡方式
