## 產品定位

**一句話**：「找到台灣任何宗教場所，了解它的一切」

全台最完整的宗教場所搜尋引擎，同時服務信眾的日常需求和旅客的文化探索。

### 關鍵決策

| 面向 | 決策 |
|------|------|
| 產品類型 | 資訊豐富的專業目錄 + 輕度社群互動 |
| 地理市場 | 台灣（可從特定縣市開始） |
| 宗教涵蓋 | 全宗教（道教/佛教/基督教/天主教/伊斯蘭教/其他） |
| 目標使用者 | 信眾（日常工具）+ 觀光客（文化探索） |
| 商業模式 | 待驗證（先做產品） |
| MVP 形式 | Web App 優先，之後再出 App |
| 產品方向 | 資料驅動目錄（先建最完整資料庫，再逐步加入文化內容與社群功能） |

## MVP 核心功能

### 1. 搜尋與篩選
- 依地區（縣市/鄉鎮）、宗教類型、主祀神明/特色篩選
- 支援關鍵字搜尋（場所名稱、地址、神明名）
- 地圖模式 + 列表模式切換

### 2. 場所詳細頁
- 基本資訊：地址、電話、開放時間、交通方式
- 照片輪播（先從 Google Maps 抓取，之後開放使用者上傳）
- 宗教類型、主祀神明、建廟年份等結構化資料

### 3. 附近探索
- 用 GPS 定位顯示附近的宗教場所
- 依距離排序，適合觀光客現場使用

### 4. 基本互動（輕社群）
- 使用者評分（1-5 星）
- 簡短評語/心得
- 「收藏」功能（需登入）

### 刻意不做的（YAGNI）
MVP 不含：線上捐獻、活動日曆、論壇社群、App 版本、多語言

## 資料策略

### 資料來源（三層）

**第一層：自動抓取（MVP 核心）**
- **Google Maps via Outscraper**：名稱、地址、電話、營業時間、評分、照片、GPS 座標（Outscraper 價格最便宜、Google Maps 是其專長）
- **內政部全國宗教資訊系統 via Firecrawl**：政府開放資料，含主祀神明、建立年份、負責人等 Google Maps 沒有的結構化資料
- 兩個來源透過名稱 + 地址比對合併，建立最完整的單一紀錄

**第二層：資料強化（上線後逐步）**
- 使用者回報錯誤/補充資訊
- 場所管理者自行認領、更新資料

**第三層：未來擴展**
- 使用者上傳照片
- 編輯團隊撰寫文化內容

### 資料規模估算
- 台灣登記寺廟：~12,000 間
- 教會/教堂：~4,500 間
- 其他宗教場所：~500 間
- **MVP 目標：涵蓋 15,000+ 場所的基本資料**

### 資料更新策略
- 每月自動重新抓取一次，比對差異更新
- 使用者回報即時處理
- 優先更新高瀏覽量的場所頁面

## 技術架構

### 前端（Web App）
- **Next.js**：SSR/SSG 混合
  - 場所頁面：靜態生成（SSG）→ SEO 最佳化
  - 搜尋結果：伺服器端渲染（SSR）
- 地圖元件：**Leaflet**（免費開源）或 Google Maps JS API
- RWD 響應式設計，手機體驗優先

### 後端 + 資料庫
- **Next.js API Routes** 處理搜尋、篩選、使用者互動
- **PostgreSQL + PostGIS**：地理空間查詢（「附近 3 公里內的廟宇」）
- **Supabase** 作為 BaaS：免費額度慷慨、內建 Auth、PostGIS 支援、即時 API

### 資料抓取 Pipeline
- **Outscraper API** → Google Maps 資料 → JSON
- **Firecrawl** → 內政部宗教網站 → Markdown → 解析
- **Python 清洗腳本**（沿用 google-maps-scraper 架構）→ 比對合併 → 寫入 PostgreSQL

### 部署
- **Vercel**（Next.js 原生支援、免費額度夠 MVP）
- **Supabase Free Tier**（500MB DB、50,000 月活用戶）

## SEO 與成長策略

### URL 結構
```
/temples/台北市/龍山寺          → 個別場所頁
/temples/台北市                 → 縣市列表頁
/temples?religion=道教          → 篩選結果頁
/explore/附近                   → 附近探索頁
```

### 每個場所頁的 SEO 要素
- 標題：`{場所名} - {縣市}{鄉鎮} | {宗教類型} | 台灣宗教指南`
- 結構化資料（Schema.org `PlaceOfWorship`）
- 自動生成的簡介段落（地址、主祀神明、特色）

### 長尾關鍵字機會
- 「龍山寺開放時間」「行天宮怎麼去」→ 個別場所頁攔截
- 「台北拜月老廟宇推薦」「高雄媽祖廟」→ 篩選列表頁攔截
- 15,000+ 頁面 × 長尾關鍵字 = 大量自然流量入口

### 初期成長（不花廣告費）
1. SEO 為主：靜態生成 15,000 頁，等 Google 索引
2. 社群分享：場所頁一鍵分享到 LINE/Facebook
3. Google Business Profile：使用者搜到你的更正版本

### 關鍵指標（MVP 上線後 3 個月目標）
- 索引頁面數：10,000+
- 月自然搜尋流量：5,000+ visits
- 場所資料覆蓋率：全台 80%+

## MVP 開發階段

### Phase 0：資料建設（1-2 週）
- 用 Outscraper 抓取全台宗教場所的 Google Maps 資料
- 用 Firecrawl 爬取內政部宗教資訊系統
- Python pipeline 清洗、比對、合併兩個來源
- 匯入 Supabase PostgreSQL（含 PostGIS）
- **交付物**：15,000+ 筆乾淨的場所資料

### Phase 1：核心搜尋體驗（2-3 週）
- Next.js 專案建立、Supabase 連接
- 場所列表頁（SSR）：篩選、排序、分頁
- 場所詳細頁（SSG）：所有結構化資訊 + 照片
- 關鍵字搜尋功能
- RWD 手機優先切版
- **交付物**：可搜尋、可瀏覽的完整目錄網站

### Phase 2：地圖與定位（1-2 週）
- 地圖模式（Leaflet + PostGIS 查詢）
- GPS 附近探索功能
- 地圖 / 列表模式切換
- **交付物**：「附近有什麼」完整體驗

### Phase 3：使用者互動（1-2 週）
- Supabase Auth 登入（Google / LINE Login）
- 場所評分 + 簡短評語
- 收藏功能
- **交付物**：輕社群互動上線

### Phase 4：SEO 優化與上線（1 週）
- Schema.org 結構化資料
- Sitemap 自動生成
- Meta tags / OG tags 優化
- Google Search Console 提交
- **交付物**：正式上線、開始累積搜尋流量
